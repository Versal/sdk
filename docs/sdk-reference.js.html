SDK Reference

Commands

init

Generates all the files needed to build a gadget in the current directory.

versal create <gadget_name> [<gadget_name_1>, <gadget_name_2>, ...]

Examples:

# create a single gadget
versal create flashcard_gadget

# create multiple gadgets
versal create physics_speed physics_position physics_gravity

The default gadget has the following structure:

├── README.md
├── assets
│   └── icon.png
├── gadget.css
├── gadget.js
├── index.html
├── manifest.json
├── package.json
├── scripts
│   └── random.js
├── sdk
│   [...]
└── test
    ├── another_spec.js
    └── gadget_spec.js

preview

Runs a local server for previewing and manually testing your gadget:

versal preview [<gadget_dir>] [<gadget_dir_1>, <gadget_dir_2>, ...]

Examples:

# equivalent to versal preview .
versal preview

# preview single gadget
versal preview ../physics_speed

# preview multiple gadgets
versal preview ./physics_speed ./physics_position ./physics_gravity

# preview all gadgets in a dir
versal preview ./physics_gadgets/*

UI

The UI simulates the environment of the Course Player that your final gadget will run in.

publish

Publish yours gadget to the Versal platform for review:

versal publish [<gadget_dir>] [<gadget_dir_1>, <gadget_dir_2>, ...]

Examples:

# equivalent to versal publish .
versal publish

# publish single gadget
versal publish ../physics_speed

# publish multiple gadgets
versal publish ./physics_speed ./physics_position ./physics_gravity

# publish all gadgets in a dir
versal publish ./physics_gadgets/*
