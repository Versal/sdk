Concepts

Core ideas

Interactive education
At Versal, we believe in the power of interactivity to change education. Learning is not necessarily about passively consuming content from video lectures and old textbooks. Learning can be so much more engaging, and fun! In order to best serve the Versal platform as a developer, you must consider education as a very exploratory process where guided exploration is encouraged for learners. After all, everyone learns in a different way and at a different pace.

Building Blocks
The building blocks from which Versal courses are constructed are gadgets. These gadgets are chunks of JavaScript code floating around, doing their thing in pretty containers. Gadgets appear in every single course a user can see. A gadget's functionality can range from holding text, images, or video, to explaining a very specific concept.

What should my gadget do?
Well, that's largely up to you. Versal has built many gadgets that enable authors to use text, images, and video in their courses. You will also find gadgets that allow users to manipulate 3D models, write out mathematical expressions, as well as conduct surveys and explore interactive geometry. This is is just scratching the surface of what a gadget can, and should do. Your gadget should serve the purpose of helping a learner understand a concept in a way that static pictures in a textbook cannot. A gadget can offer the learner a playground in which they can explore concepts using mouse, keyboard, audio, and touch interactions. The key to designing a great gadget is to consider the learner's experience of interacting with your gadget.

Some Considerations

Consider the following design principles while creating your gadgets:
  • Intuitive controls for your simulation, game, or visualization should require little explanation for users to feel comfortable with them.
  • Build an interface for author control into your gadget. While Versal provides external property sheets to allow authors to configure your gadget, a fluid interface between what an author sees and what a learner sees when viewing your gadget is something to strive for.
  • Sometimes, the more control the learner has within your gadget, the more the learner will be encouraged to experiment while achieving a greater understanding of the material.
  • Think outside the box, because gadgets that fill 'holes' in the educational tapestry are sorely needed. You can fill these holes for current and future course authors.
  • Conceptual gadgets can be used to test the learner's progress and understanding of course material. There is more to life than multiple-choice quizzes.

Roles

Author
The creator of Versal courses

Learner
The end user of Versal courses

Developer
You! (or any gadget developer)

SDK

Command line tool for creating, previewing and publishing gadgets

Course player

The user interface that authors and learners use to interact with Versal courses

Course

A collection of lessons created by an author

Lesson

A collection of gadgets chosen and configured by a course author

Gadget

An browser application (JS/HTML/CSS/etc) that can be configured by an author and then presented in a course for teaching something to a learner. Gadgets can be simulations, flash cards, charts, visualizations, etc. Gadgets tend to be highly visual and interactive but can also be completely invisible and simply offer a service to other gadgets in the course.

Gadget project
Source code (HTML, CSS, Javascript, etc) of a gadget

Gadget instance
A gadget that has been added to a course's lesson by an author. Most gadgets will have an authoring mode for configuring the gadget instance for use by learners.

Asset

A resource file used by a gadget project or instance (e.g. image, video, spreadsheet). Assets can be bundled with a gadget project and/or chosen by an author through a provided asset picking mechanism in the course player.
